🧩 组件清单 Components

📌 统计状态栏

展示总物品数、最爱总数、讨厌总数三个徽章。数据从 items_data.js 自动计算。

静态auto-compute

🔍 搜索框

输入关键词实时过滤物品列表。匹配物品的 name 字段（中文名）。

实时过滤input事件

🏷️ 分类标签页

19 个分类按钮 + "全部"。点击切换 activeCategory，重新渲染商品网格。

19分类切换过滤

🃏 商品卡片

展示图片、中文名、分类标签、来源、价格、星级选择器、数量加减及小计。点击进入详情弹窗。

响应式网格星级联动

⭐ 星级选择器

仅对有 has_quality 的物品（作物、季节采集物）显示。选项：普通/银星/金星/铱星，切换时价格联动更新。

品质价格联动

🔢 数量控制器

− / 数字输入 / + 三部件。范围 0~999。变更时更新小计和购物车状态。

加减按钮手动输入

🛒 购物车栏

显示当前总件数、总金额、结账按钮、清空按钮。任何数量变动后自动更新。

实时总计浮动挂载

💬 物品详情弹窗

点击卡片打开。展示完整信息（品质选择、添加数量 + 加入购物车）、村民赠送喜好表。

模态框5级好感

🧾 结账弹窗

表格展示购物车所有条目：序号、商品、数量、单价、小计、最爱数、讨厌数。底部汇总总额。

表格明细确认清空

📢 Toast 提示

底部弹出短暂提示：成功加入购物车 / 结账成功 / 清空购物车。2秒自动消失。

通知2s消失

🔄 用户交互流程 UX Flow

🌱 打开页面 → 加载 items_data.js（374 件物品） → 渲染分类按钮 + 商品网格
🔍 搜索 / 点分类 → 实时过滤商品列表 → 重新渲染网格
🖱️ 调整数量 +/- → 更新购物车数组 → 同步小计显示 + 购物车栏
⭐ 切换品质 → 更新单价显示 → 联动小计 + 购物车价格
📦 点击商品卡片 → 打开详情弹窗 → 查看村民喜好 + 加入购物车
✨ 点击结账 → 打开订单明细弹窗 → 确认 → 清空购物车 → 提示

💾 数据架构 Data Flow

items_data.js  // 全局变量 window.ITEMS_DATA (374 items)
│
├── item {
│    id, name, category, price, has_quality,
│    qualities: { 普通, 银星, 金星, 铱星 },
│    image, source, description,
│    gifting: { love[], like[], neutral[], dislike[], hate[] }
│   }
│
├── categories[]  // 19 个分类
│
├── cartItems[]  // 购物车状态（运行时）
│   { itemId, name, quality, quantity, price, ... }
│
└── render()  // 根据 filteredItems 绘制网格
